# 2단 레이아웃 구조 변경 및 공지사항 제거 계획

현재의 3단 구조(1단: 키워드, 2단: 질문 목록, 3단: 공지사항 및 상세 내용)를 단순하고 직관적인 **2단 구조**로 변경하고, 사용되지 않는 **공지사항 영역을 완전히 제거**합니다.

## User Review Required
> [!IMPORTANT]
> **화면 전환 방식 설계**: 2단 구조가 되면 화면은 다음과 같이 작동합니다.
> - **1단 (좌측)**: 기존과 동일한 '키워드 필터' 사이드바.
> - **2단 (우측 메인)**: 
>   - 기본 상태에서는 질문 등록 폼과 **'질문 목록'**이 보입니다.
>   - 질문 카드를 클릭하면, 우측 화면 전체가 해당 질문의 **'상세 내용 및 답변 작성 스레드'**로 전환됩니다.
>   - 상세 화면에서 '← 뒤로 가기'를 누르면 다시 질문 목록으로 돌아옵니다.
> 
> 이와 같은 작동 방식이 원하시는 구조가 맞는지 확인 부탁드립니다! 맞다면 승인해 주시면 즉시 코딩을 시작하겠습니다.

## Proposed Changes

### 1. HTML 구조 변경 (`index.html`)
- 기존의 3단 영역을 담던 `<section class="sidebar-details">`를 삭제합니다.
- 내부의 `notice-view`(공지사항) 관련 HTML 코드를 완전히 삭제합니다.
- `thread-view`(질문 상세 및 답변 영역)를 기존의 2단 영역이었던 `main-feed`와 같은 레벨 또는 그 내부에 배치하여, 목록과 상세 화면이 서로 교대로 보이도록(토글) 구조를 변경합니다.

### 2. CSS 레이아웃 변경 (`styles.css`)
- `.app-container`의 CSS Grid 설정을 기존 `240px 1fr 380px`에서 **`240px 1fr`**(2단 구조)로 단순화합니다.
- 공지사항 관련 스타일 클래스(`.notice-panel`, `.notice-card`, `.notice-badge` 등)를 모두 삭제하여 코드를 최적화합니다.
- 질문 목록 컨테이너와 질문 상세 컨테이너가 우측 메인 영역을 자연스럽게 꽉 채워 사용하도록 스타일을 조정합니다.

### 3. JavaScript 로직 변경 (`app.js`)
- 화면 렌더링 로직(`renderRightPanel` 등)을 수정합니다.
- 공지사항을 띄우는 로직을 제거합니다.
- `state.activeQuestionId`의 유무에 따라, 질문이 선택되지 않았을 때는 우측 화면에 **질문 피드(목록)**를 보여주고, 질문이 선택되었을 때는 **질문 상세 스레드**를 보여주는 화면 전환 로직을 구현합니다.

## Verification Plan
1. **공지사항 삭제 확인**: 우측에 항상 고정되어 있던 알림판 영역이 완전히 사라졌는지 확인합니다.
2. **2단 구조 확인**: 브라우저 화면이 좌측(키워드)과 우측(메인)의 두 부분으로만 나뉘어 있는지 검증합니다.
3. **화면 전환 동작 확인**: 우측 메인 영역에서 질문을 클릭했을 때 질문 상세 내용과 댓글 창으로 잘 바뀌는지, '뒤로 가기' 버튼을 눌렀을 때 다시 전체 질문 목록으로 잘 돌아오는지 확인합니다.
